FreeRDP is a free, Apache-licensed implementation of the Remote Desktop Protocol (RDP). It provides a full C library (`libfreerdp`) for embedding RDP connectivity into applications, ready-to-run client frontends for every major platform, and server-side components including a screen-sharing shadow server and a programmable RDP proxy — all backed by the WinPR Windows Portability Runtime.

## What is FreeRDP?

FreeRDP implements the Microsoft Remote Desktop Protocol as an open standard, giving developers full control over RDP connections. The project ships several distinct components:

<CardGroup cols={3}>
  <Card title="libfreerdp" icon="book">
    Core protocol library providing codecs, crypto, GDI, channels, and the settings system.
  </Card>
  <Card title="Client Frontends" icon="window">
    Platform-specific clients: xfreerdp (X11), sdl-freerdp (SDL2/3), wlfreerdp (Wayland), Windows, macOS, Android, iOS.
  </Card>
  <Card title="WinPR" icon="layer-group">
    Windows Portability Runtime — a cross-platform Win32 API compatibility layer used by FreeRDP and standalone projects.
  </Card>
  <Card title="Shadow Server" icon="eye">
    Screen-sharing RDP server that captures and streams the local desktop to remote clients.
  </Card>
  <Card title="RDP Proxy" icon="shuffle">
    Pluggable man-in-the-middle proxy for inspecting, logging, and modifying RDP sessions.
  </Card>
  <Card title="Virtual Channels" icon="plug">
    30+ channel plugins: clipboard, audio, drive/printer/smartcard redirection, USB, graphics pipeline, camera, and more.
  </Card>
</CardGroup>

## Quick Start

<Steps>
  <Step title="Install dependencies and build">
    Install OpenSSL, CMake ≥ 3.13, and any optional codec libraries, then build with CMake.

    ```bash
    cmake -B build -DCMAKE_BUILD_TYPE=Release -DWITH_X11=ON
    cmake --build build --parallel
    sudo cmake --install build
    ```
  </Step>
  <Step title="Connect to an RDP server">
    Use `xfreerdp` to open a connection. Supply `/v:` for the host and `/u:` for the username.

    ```bash
    xfreerdp /v:192.168.1.100 /u:Administrator /p:Password123
    ```
  </Step>
  <Step title="Integrate the library">
    Include `<freerdp/freerdp.h>` and link against `libfreerdp3` to embed RDP in your own application.

    ```c
    #include <freerdp/freerdp.h>

    freerdp* instance = freerdp_new();
    freerdp_context_new(instance);
    // configure settings, set callbacks, then:
    freerdp_connect(instance);
    ```
  </Step>
</Steps>

## Key Features

- **Full RDP protocol coverage** — NLA, TLS, RDP security, Kerberos, FIDO2/WebAuthn authentication
- **Hardware-accelerated codecs** — H.264 via OpenH264 or FFmpeg, RemoteFX, NSCodec, ClearCodec
- **Rich virtual channels** — clipboard sync, audio in/out, drive/printer/smartcard/USB redirection, graphics pipeline (GFX), camera
- **Multi-platform** — Linux (X11, Wayland), Windows, macOS, Android, iOS, BSD
- **Embedded-ready** — Clean C99 API, Apache 2.0 license, no proprietary dependencies required
- **WinPR** — optional standalone portability runtime covering threads, crypto, registry, named pipes, and more

<Note>
  FreeRDP 3.x is the current stable series. The API has breaking changes from 2.x — consult the [security and versioning reference](/reference/security) when upgrading.
</Note>
